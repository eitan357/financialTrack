### Task 3: Fix categories.ts dynamic import

`updateCategory` uses `await import('firebase/firestore')` inside the function body — an async dynamic import — while all other firestore files use top-level static imports. This is inconsistent and bypasses bundler tree-shaking. `updateDoc` and `doc` are already available from firebase/firestore; they just need to be added to the existing static import at the top of the file.

**Files:**
- Modify: `src/lib/firestore/categories.ts`

- [ ] **Step 1: Read current import line**

Current `src/lib/firestore/categories.ts` line 1-5:
```ts
import {
  getFirestore, collection, getDocs, addDoc, doc, deleteDoc,
  query, limit, where, writeBatch, runTransaction,
} from 'firebase/firestore'
```

`updateDoc` is missing from the static import.

- [ ] **Step 2: Update the static import**

Replace the import block at the top of `src/lib/firestore/categories.ts`:

```ts
import {
  getFirestore, collection, getDocs, addDoc, doc, updateDoc, deleteDoc,
  query, limit, where, writeBatch, runTransaction, setDoc,
} from 'firebase/firestore'
```

- [ ] **Step 3: Fix updateCategory to use static imports**

Current `updateCategory` (lines 31-35):
```ts
export async function updateCategory(id: string, updates: Partial<Omit<Category, 'id'>>): Promise<void> {
  appCache.del(CACHE_KEY)
  const { doc: firestoreDoc, updateDoc } = await import('firebase/firestore')
  await updateDoc(firestoreDoc(getDb(), 'categories', id), updates)
}
```

Replace with:
```ts
export async function updateCategory(id: string, updates: Partial<Omit<Category, 'id'>>): Promise<void> {
  appCache.del(CACHE_KEY)
  await updateDoc(doc(getDb(), 'categories', id), updates)
}
```
