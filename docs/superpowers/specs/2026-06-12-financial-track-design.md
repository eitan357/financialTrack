## 1. רקע ומטרה

המשתמש מנהל כרגע מעקב פיננסי מורכב בגיליון Excel — הוצאות, הכנסות, השקעות ודיבידנדים.
התהליך דורש הרבה עבודה ידנית: העתקת עסקאות אשראי, קיטלוג ידני, השוואות יתרות בנק, וחישובי מטבע.

**מטרה:** להפוך את תהליך המעקב לאפליקציה שמבצעת את הפעולות הידניות אוטומטית.

### כאבים שיש לפתור
1. העתקה ידנית של CSV/XLSX אשראי → ייבוא אוטומטי עם קיטלוג חכם הניתן לעריכה
2. קיטלוג ידני של כל עסקה → קיטלוג אוטומטי לפי חוקי מילות מפתח + היסטוריה, עם אפשרות ביטול
3. השוואת יתרות בנק ידנית → הזנת יתרה בפועל + חישוב אוטומטי של יתרה צפויה (בתאריך גמיש)
4. חישוב דיבידנדים במטבעות שונים → טופס מובנה עם תמיכה בכל מטבע
5. מעקב השקעות מבלבל (לא הוצאה, לא הכנסה) → ספר חשבונות נפרד
6. ניכויי משכורת משתנים כל חודש → טופס עריכה מלאה בכל הזנה
7. אין ייצוא נוח לאקסל → ייצוא מכל טבלה בלחיצה אחת

## 3. ישויות נתונים (Firestore)

### `accounts` — חשבונות וכרטיסי אשראי
ניתן להוסיף / לערוך / למחוק.
```
id, name, type (credit|bank|cash), last4digits?, color, isActive
```
**ברירות מחדל:**
- אשראי בהצדעה (credit, 8729)
- אשראי One Zero (credit)
- בנק One Zero (bank)
- בנק לאומי (bank)
- מזומן (cash)

### `categories` — קטגוריות הוצאות
ניתנות להוספה / עריכה / מחיקה. כוללות נקודת ייחוס חודשית (לא מגבלה).
```
id, name, monthlyTarget?, color, isActive
```
**ברירות מחדל:** אוכל, חשבונות, רפואה, תחבורה, תחזוקה, חופשים,
מתנות לעצמי, מתנות לאחרים, קורסים וחוגים, סופש כיף, בילויים, בגדים, אוכל בחוץ, השקעות, שכירות

### `categorization_rules` — חוקי קיטלוג אוטומטי ← **חדש**
מנהלים את הלוגיקה של הקיטלוג החכם. ניתנים להוספה / עריכה / מחיקה בהגדרות.
```
id, keyword (string), matchType (contains|exact|startsWith), categoryId, priority (number), createdAt
```
**דוגמאות:**
- "שופרסל" → קטגוריה: אוכל
- "YES" → קטגוריה: חשבונות
- "פארם" → קטגוריה: רפואה

**היסטוריה אוטומטית:** כל פעם שמשתמש בוחר קטגוריה לעסקה ידנית — המערכת מציעה לשמור חוק חדש.

### `transactions` — עסקאות
```
id, date, merchantName, description?, amount, currency, accountId, categoryId?,
source (csv_import|xlsx_import|manual), isImmediate (חיוב מידי),
month (YYYY-MM), originalFileName?
```

### `salary_entries` — משכורת חודשית
כל שדה ניתן לעריכה בכל חודש — הניכויים משתנים מחודש לחודש.
```
id, month (YYYY-MM), employerName, grossAmount,
deductions: { incomeTax, nationalInsurance, healthInsurance, pension, trainingFund },
netAmount, notes?
```
**הערה:** שם המעסיק ניתן לעריכה — מתחלף כשמשנים עבודה. הטופס מציע ערכים מהחודש הקודם לנוחות אך ניתן לשנות הכל.

### `income_entries` — הכנסות נוספות
גמיש לחלוטין — מילואים, עמלות, בונוס, החזר, יד2, כל מה שיהיה.
```
id, month (YYYY-MM), sourceName, amount, date, notes?
```

### `investment_types` — סוגי השקעות
ניתן להוסיף / לערוך / למחוק.
```
id, name, currency (ILS|USD|EUR|...), notes?
```
**דוגמאות:** MSTY (USD), Headstart — סופר Fair (ILS)

### `investment_entries` — פעולות השקעה
העברת כסף להשקעות — לא הוצאה ולא הכנסה, ספר נפרד.
```
id, date, investmentTypeId, amount, currency, notes?
```

### `investment_conversions` — המרות מטבע בהשקעות ← **חדש**
כאשר מוכרים חלק מהשקעה ומקבלים שקלים בחשבון הבנק.
```
id, date, investmentTypeId,
ilsReceived (סכום שנתקבל בחשבון בנק),
foreignAmountReduced? (כמה יחידות/דולרים הופחתו),
notes?
```
**לוגיקה:** המשתמש מזין את הסכום בשקלים שהגיע לחשבון. לחלופין יכול לציין כמה USD/אחר הופחת מהסכום בהשקעה. **אין חישוב שער חליפין אוטומטי** — המשתמש מזין את מה שראה בבנק.

### `dividends` — דיבידנדים
נחשבים **הכנסה** בחישוב החיסכון החודשי. תמיכה בכל מטבע.
```
id, month (YYYY-MM), investmentTypeId,
amount (הסכום המקורי),
currency (USD|EUR|ILS|GBP|...) ← כל מטבע
ilsEquivalent? (אם המשתמש מזין ידנית),
date, notes?
```
**הערה:** לא מחשבים שער חליפין אוטומטי. המשתמש מזין את הסכום במטבע המקורי וכן (אופציונלי) את הסכום בשקלים שקיבל בפועל מהבנק.

### `bank_reconciliations` — אימות יתרות בנק
```
id, month (YYYY-MM), accountId, actualBalance, expectedBalance,
date (גמיש — נקבע לפי תאריך כניסת המשכורת, לא קבוע ל-3 בחודש),
notes?
```

### `monthly_settings` — נקודות ייחוס חודשיות
```
id, month (YYYY-MM), categoryTargets: { categoryId → amount }
```

## 4. חישוב חיסכון חודשי

```
חיסכון = הכנסות - הוצאות
הכנסות = משכורת נטו + הכנסות נוספות + דיבידנדים (בשקלים)
הוצאות = עסקאות אשראי + עסקאות בנק + מזומן + שכירות
השקעות = ספר נפרד (מורידות יתרת בנק אך אינן הוצאה)
```

**אימות יתרת בנק (תאריך גמיש):**
```
יתרה_צפויה = יתרה_חודש_קודם + משכורת + הכנסות_נוספות
             - סה"כ_אשראי_חויב - שכירות - הוצאות_בנק_ישיר - להשקעות
יתרה_בפועל = מה שהמשתמש הכניס ידנית (מסתכל בבנק)
הפרש = יתרה_בפועל - יתרה_צפויה (אמור להיות ≈ 0)
```

**השוואה היא ידנית:** המשתמש מסתכל בבנק ומזין את היתרה. האפליקציה מחשבת את הצפוי ומראה פער. אין חיבור אוטומטי לבנק.

## 5. מסכים

### מסך 1 — ראשי (Dashboard)
- בוחר חודש בראש המסך
- 4 כרטיסי סיכום: הכנסות / הוצאות / חיסכון / להשקעות
- Progress bars לפי קטגוריה: בפועל vs נקודת ייחוס (אדום = עברת, ירוק = בטווח)
- כרטיס אימות יתרת בנק — ✓ תואם / ✗ פער (כפתור "אמת עכשיו" → מסך אימות)
- כרטיס דיבידנדים החודש עם כפתור הוספה מהיר
- כפתור ⚙ להגדרות בפינה

### מסך 2 — עסקאות
- פילטר pills: הכל / לפי חשבון / לפי קטגוריה / ⚠ ללא קטגוריה
- שורת עסקה: שם בית עסק, תאריך, תג חשבון, תג קטגוריה, סכום
- עסקאות ללא קטגוריה מסומנות בגבול כחול
- חיובים מיידיים (PayPal, מטבע חוץ) עם תג "מיידי"
- **לחיצה על עסקה → עריכת קטגוריה + תיאור ← אפשרות לביטול/שינוי קיטלוג אוטומטי**
- כפתור "ייצוא לאקסל" בראש הטבלה

### מסך 3 — ייבוא חודשי
אשף 5 שלבים שמנחה את התהליך הידני הנוכחי:

**שלב 1 — אשראי בהצדעה (CSV או XLSX)**
- כפתור העלאת קובץ (תומך בפורמטים: `.csv`, `.xlsx`)
- **אם XLSX — בחירת גליון** מתוך הגליונות הנמצאים בקובץ (לרוב: "פירוט" + "קניות בחול")
  - ניתן לייבא גליון אחד, כמה גליונות, או כולם בבת-אחת
- תצוגה מקדימה של עסקאות לפני אישור
- **קיטלוג אוטומטי** (ראה סעיף 6)
- עסקאות חדשות → בקשת קטגוריה + הצעה לשמור חוק חדש
- הקובץ המקורי **לא נשמר** — רק העסקאות המעובדות נכנסות ל-Firestore

**שלב 2 — אשראי One Zero (CSV/XLSX)** — אותו flow

**שלב 3 — משכורת**
- שם מעסיק (נשמר מהחודש הקודם אך ניתן לעריכה)
- ברוטו
- ניכויים — **כל שדה ניתן לעריכה בכל חודש** (הזנה ידנית, לא מקובץ):
  - מס הכנסה
  - ביטוח לאומי
  - ביטוח בריאות
  - פנסיה
  - קרן השתלמות
- = נטו (מחושב אוטומטית)

**שלב 4 — הכנסות נוספות**
- רשימה דינמית — כפתור "+ הוסף הכנסה"
- כל פריט: שם מקור (dropdown מהמקורות שהוגדרו + "אחר" עם שדה חופשי), סכום, תאריך
- מתאים ל: מילואים (3 תשלומים נפרדים), עמלות, בונוס, החזרים

**שלב 5 — מזומן**
- הזנת הוצאות מזומן ידנית

### מסך 4 — השקעות ודיבידנדים
- סיכום פורטפוליו: סה"כ הועבר להשקעות (לפי מטבע, ללא המרה אוטומטית)
- רשימת סוגי השקעות עם יתרות (ניתנים להוספה/מחיקה)
- כפתור "+ העברה להשקעה" — בוחר סוג, מכניס סכום
- **כפתור "+ המרה/משיכה" ← חדש:**
  - בוחר סוג השקעה
  - מזין: סכום ILS שהתקבל בחשבון הבנק
  - בחירה אופציונלית: הפחת X מהסכום בהשקעה (בדולרים/מטבע הרלוונטי)
  - **לא מחשבים שער חליפין** — המשתמש מזין מה שראה בבנק
- כפתור "+ דיבידנד":
  - בוחר מקור (מהרשימה המוגדרת)
  - בוחר מטבע (USD, EUR, ILS, GBP, ועוד)
  - מכניס סכום במטבע המקורי
  - שדה "סכום בשקלים שהתקבל" (אופציונלי — לחישוב חיסכון)
  - דיבידנד נספר כהכנסה בחישוב החיסכון לפי סכום ה-ILS שהוזן
- כפתור "ייצוא לאקסל" בראש הטבלה

### מסך 5 — דוחות
שלושה טאבים, כל אחד עם כפתור "ייצוא לאקסל":
- **חודשי** — גרף עמודות של חיסכון חודשי (ירוק/אדום)
- **שנתי** — טבלת קטגוריות עם צפי vs בפועל לכל החודשים, סה"כ שנתי
- **אימות** — הכנסת יתרת בנק One Zero + לאומי בתאריך נבחר (גמיש), השוואה לצפוי, הצגת הפרש

### מסך 6 — הגדרות
- **חשבונות** — רשימת כרטיסי אשראי וחשבונות בנק (הוסף / ערוך / מחק)
- **קטגוריות** — קטגוריות הוצאות עם נקודות ייחוס חודשיות (הוסף / ערוך / מחק)
- **חוקי קיטלוג ← חדש** — ניהול חוקי מילות מפתח (הוסף / ערוך / מחק)
  - מציג: מילה / ביטוי → קטגוריה
  - ניתן לשנות עדיפות (בחוקים מתנגשים)
- **מקורות הכנסה** — שמות מקורות שמופיעים כאפשרויות בשלב 4 (הוסף / מחק). לא מגבילים — תמיד אפשר להכניס שם חופשי.
- **סוגי השקעות** — MSTY, Headstart וכו' (הוסף / ערוך / מחק)

## 6. ייבוא CSV/XLSX — לוגיקה

### פורמטים נתמכים
- `.csv` — ייבוא ישיר
- `.xlsx` — פתיחת Excel עם בחירת גליון. הגליונות הנפוצים בדוחות Discount Bank / Max:
  - **"פירוט"** — חיובים רגילים
  - **"קניות בחול"** — עסקאות במטבע חוץ (AliExpress, Amazon וכו')
  - ניתן לייבא אחד או שניהם בבת-אחת

### עמודות פורמט Discount Bank / Max
תאריך עסקה, שם בית העסק, קטגוריה (של הבנק), 4 ספרות אחרונות, סוג עסקה, סכום חיוב, מטבע חיוב, סכום מקורי, מטבע מקורי, תאריך חיוב, הערות

### קיטלוג אוטומטי — שתי שכבות
**שכבה 1 — חוקי מילות מפתח** (עדיפות גבוהה יותר):
- בדיקה אם שם בית העסק מכיל / מתחיל ב / תואם מדויק לאחת ממילות המפתח המוגדרות
- אם כן → קטגוריה לפי החוק

**שכבה 2 — היסטוריה:**
- אם אין חוק תואם — חיפוש בעסקאות קודמות לאותו `merchantName`
- אם קיים → קטגוריה מהעבר

**אם לא נמצאה התאמה:**
- מסומן "ממתין לקיטלוג" (גבול כחול)
- מציג הצעה מהקטגוריה של הבנק בקובץ
- לאחר בחירת המשתמש → מציע "לשמור חוק: מילה X → קטגוריה Y?"

### חיובים מיידיים
עסקאות עם `סוג עסקה = "חיוב עסקות מיידי"` → `isImmediate=true`, תג "מיידי" בממשק.

## 7. ייצוא לאקסל

כל טבלה בממשק תכלול כפתור "ייצוא":
- **פורמט:** XLSX (פותח ישירות ב-Excel ו-Google Sheets)

- **שמות עמודות בעברית**

  - מסך עסקאות (כל הפילטרים)
  - מסך השקעות ודיבידנדים
  - דוח חודשי
  - דוח שנתי
  - דוח אימות יתרות

## 8. גיבויים ומה נשמר

### מה נשמר
**נשמר:** הנתונים המעובדים בלבד — עסקאות, משכורות, הכנסות, דיבידנדים, השקעות — כולם נכנסים ל-Firestore.

**לא נשמר:** קבצי CSV/XLSX המקוריים שמועלים לייבוא. הקובץ מעובד בזיכרון, הנתונים מחולצים, והקובץ מושלך.

### Firestore — אמינות
Firebase Firestore מספק redundancy אוטומטי ב-3 אזורים גיאוגרפיים. על Blaze plan קיים Point-in-Time Recovery (30 יום אחורה).

### ייצוא ידני כגיבוי
ניתן לייצא כל טבלה לאקסל בכל עת — זה משמש כגיבוי "ידידותי" של הנתונים.
